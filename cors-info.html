MP4 Converter - CORS Information

Why is the converter not working?

Our MP4 converter uses advanced browser technologies (WebAssembly with SharedArrayBuffer) that require specific security settings. GitHub Pages doesn't support the required HTTP headers by default.

How to use this converter:

Option 1: Run locally with a simple server that sets the proper headers:

  1. Clone the repository
  2. Install a simple HTTP server like http-server (npm install -g http-server)
  3. Run with the CORS headers: http-server --cors '*' -p 8080
  4. Open http://localhost:8080 in your browser

Option 2: Use a service that supports the required headers:

  • Deploy to Netlify, Vercel, or another service that allows custom headers
  • Set up the headers:
    • Cross-Origin-Embedder-Policy: require-corp
    • Cross-Origin-Opener-Policy: same-origin
